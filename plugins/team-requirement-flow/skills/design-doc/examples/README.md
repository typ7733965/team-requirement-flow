# 设计方案示例

本目录是 `design-doc` Skill 的「真实产出示例」，来自 game-operation 项目「用户端登录认证组件」需求。

## 文件

- `用户端登录认证组件设计方案.md` —— 完整 9 节设计方案

## 阅读要点

参考以下章节如何写：

1. **第二节 整体架构** —— ASCII 时序图 + 模块依赖图，标注每个仓库的角色
2. **第三节 数据模型** —— 多租户 Redis Key 命名规则（`{app_key}:{app_uuid}` 双前缀）
3. **第五节 错误码** —— 字符串字面量（如 `"USER_GONE"`）+ 触发条件 + 调用方处理建议
4. **第六节 关键设计决策** —— 写「为什么这样做」，每条决策有候选方案 + 选择 + 理由
5. **第七节 实施步骤拆分** —— 8 个步骤，每步可独立 commit

## 注意

本示例中的具体业务（登录、SMS、token）与本插件无关，仅作为**结构与文风**的参考。
团队成员可以基于此对齐写设计方案的"调性"。
